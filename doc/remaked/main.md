# MVU (Magical-Variable-Update)

## 简介
MVU 是一个为 SillyTavern 设计的变量管理脚本，它的目的是简化角色卡制作者对于变量管理方面的工作，使创作者可以专注于人物/世界本身，更高效地完成创作。

通过这个扩展，你可以：
 - 使用 MVU 预先提供的提示词，快速地为你的角色卡添加变量，并在提示词中简单地获取它。
 - 在楼层之间维护变量状态，正确处理楼层 删除/Swipe 等逻辑。
 - 通过结构化的变量，简化角色卡前端的开发。
 - 通过 MVU 提供的各种事件，为 llm 的异常输出兜底，根据变量状态的变更自由编写逻辑。

## 为什么选择 MVU
 - 丰富的事件回调：你可以通过那些回调达成各种各样个性化的需求。
 - 充分的测试：对于变量更新的核心代码[1]，我们保证测试的行覆盖率高于 75% ，包括了各种各样的变量更新场景。
 - 符合 LLM 对齐的范式：MVU 使用类 loadsh 的函数式调用来实现变量更新，标准的 JPath 来描述路径，可以受益于现在各大 LLM 对于代码编写方面功能的对齐工作。
 - 代码编写友好的接口：对于 MVU 的 InitVar 格式及脚本用的所有接口，都提供了对应的 JSON Schema，以及类型定义 `d.ts` 文件，便于在工程化的代码编写中进行引用，获得友好的代码补全和说明。
 - 同层角色卡兼容：MVU 核心逻辑都不依赖于酒馆状态，创作者可以在同层前端中调用 MVU 对外的 事件/函数 接口进行变量的更新处理操作。

## 友情链接
在这个领域，也有其他作者在进行类似功能的开发，他们的作品各有优势，也有各自注重的方向，你可以对这些框架进行尝试，选择最适合自己的一个。
 - [小白X](https://github.com/RT15548/LittleWhiteBox)
   包含酒馆下常用功能的一个一体化解决方案，它也提供了对应的变量更新，以及常用的分阶段提示词机制。可以在 GUI 界面上操作修改，较为便利
 - [SAM](https://github.com/DefinitelyNotProcrastinating/ST_var_manager)
   这个变量管理系统注重于变量修改的便利性。它始终将变量当前状态存储于聊天记录本身中，可以方便地进行调整。此外，在任务机制上，它也进行了一定的创新。允许 LLM 规划未来的变量更新任务。
 - [ERA](https://github.com/RockingSisyphus/ERA-EfficientRollbackArchitecture)
   这个变量管理系统选择了另一种新的范式来对变量的 新增/修改/删除 机制进行描述，并专注于对聊天文件占用空间的节省，采用了变化量记录的机制，避免对变量的反复存储。




[1]: 指 `button.ts/variable_init.ts/function_call.ts/main.ts/panel.ts` 这些与酒馆直接交互的接口以外的所有文件。
