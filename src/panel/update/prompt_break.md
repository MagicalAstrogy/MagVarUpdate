# 破限方案

## 使用内置破限

负责变量更新的 AI 将由 MVU 内置的提示词破限.

感谢 @离 提供的破限提示词。主要面向 Gemini/Claude，对其他模型亦有一定的效力。

## 使用当前预设

负责变量更新的 AI 将收到预设提示词, 被预设破限.

但是预设往往规定了写作任务，因此负责变量更新的 AI 可能会选择继续剧情而不是直接更新变量, 导致其实是在推进剧情的同时分析了变量——变量的更新结果实际上是属于未来剧情的, 与当前回复并不吻合.

## 使用其他预设

负责变量更新的 AI 将收到你指定的已保存 preset 的提示词顺序和条目内容.

这个模式不会切换 SillyTavern 当前正在使用的 preset，因此不会触发 preset 切换带来的连接、回调和聊天重载开销。

需要注意：

- 这里读取的是“已保存的 preset 内容”，不是当前面板上尚未保存的 `in_use` 临时修改。
- regex 仍然沿用当前环境，不会改成目标 preset 的 regex。
- 如果你选择“自定义”模型来源，则 API 地址、模型名和温度等自定义请求参数仍以 MVU 面板中的额外模型配置为准。
